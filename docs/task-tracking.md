# Task Tracking – PopUpOslo

## Approach

Work on the project was organised using a feature-based workflow with Git.

Each major feature was developed in a separate branch and later merged into the main branch.

---

## Branches

Examples of branches used during development:
- feature/booking-system
- feature/console-ux
- fix/admin-events
- docs/final-project-documentation

---

## Task Distribution

Tasks were divided between team members based on system areas:

- Karyna: UI, event management, admin functionality, UX improvements, documentation
- Maria: booking system, event management, ticket options, validation, database, documentation

---

## Progress Tracking

Progress was tracked using:
- Git commits
- Branch history
- Completed features

---

## Collaboration

Development was done collaboratively through GitHub.

Changes were merged after completing features, and testing was performed continuously during development.
